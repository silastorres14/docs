# Visão Geral do Microsoft Entra ID

<img src="/assets/icons/identity.svg" alt="Microsoft Entra ID" className="h-12 w-12 float-right ml-4" />

O Microsoft Entra ID (anteriormente conhecido como Azure Active Directory) é o serviço de gestão de identidade e acesso baseado na nuvem da Microsoft. Serve como o sistema de identidade principal para Microsoft 365, Azure e milhares de aplicações integradas.

## Recursos Principais

O Entra ID fornece um conjunto abrangente de capacidades para gerir identidades e acessos:

  <Card title="Acesso a Aplicações" icon="apps">

    Gerir direitos de acesso, certificações e privilégios com controlo detalhado.
  </Card>
</CardGroup>

## Conceitos Fundamentais

### Tenant (Inquilino)

Um tenant do Entra ID é uma instância dedicada e isolada do serviço. Cada organização recebe seu próprio tenant ao se inscrever no Microsoft 365 ou Azure. Um tenant é identificado por um domínio inicial no formato `seudominio.onmicrosoft.com`.

```powershell
# Obter informações sobre o tenant atual usando PowerShell
Connect-MgGraph -Scopes "Organization.Read.All"
Get-MgOrganization | Select-Object Id, DisplayName, Domains
```

### Utilizadores e Identidades

No Entra ID, os utilizadores são representados como objetos com múltiplos atributos. Cada utilizador tem um identificador único (ObjectID) e normalmente um nome principal de utilizador (UPN) no formato `utilizador@dominio.com`.

<Tabs>
  <Tab title="Portal Entra ID">
    Para gerir utilizadores no portal:
    1. Aceda ao [Portal do Entra ID](https://entra.microsoft.com)
    2. Navegue para "Identidades" > "Utilizadores"
    3. A partir daqui, pode criar, editar ou desativar contas
  </Tab>
  <Tab title="PowerShell">
    ```powershell
    # Listar utilizadores com PowerShell
    Connect-MgGraph -Scopes "User.Read.All"
    Get-MgUser -Top 10 | Select-Object DisplayName, UserPrincipalName, Id
    
    # Criar um novo utilizador
    $PasswordProfile = @{
      Password = 'TemporaryP@ssw0rd!'
      ForceChangePasswordNextSignIn = $true
    }
    
    New-MgUser -DisplayName "Ana Silva" `
               -UserPrincipalName "ana.silva@contoso.com" `
               -PasswordProfile $PasswordProfile `
               -AccountEnabled $true `
               -MailNickname "anasilva"
    ```
  </Tab>
  <Tab title="Microsoft Graph">
    ```http
    # Listar utilizadores com Graph API
    GET https://graph.microsoft.com/v1.0/users
    
    # Criar um novo utilizador
    POST https://graph.microsoft.com/v1.0/users
    Content-Type: application/json
    
    {
      "displayName": "Ana Silva",
      "userPrincipalName": "ana.silva@contoso.com",
      "passwordProfile": {
        "password": "TemporaryP@ssw0rd!",
        "forceChangePasswordNextSignIn": true
      },
      "accountEnabled": true,
      "mailNickname": "anasilva"
    }
    ```
  </Tab>
</Tabs>

### Grupos e Atribuição de Acesso

Os grupos são fundamentais para gerir acessos de forma escalável. O Entra ID suporta:

- **Grupos de Segurança**: Usados para gerir acesso a recursos
- **Grupos Microsoft 365**: Proporcionam colaboração com ferramentas integradas
- **Grupos Dinâmicos**: Membros são adicionados/removidos automaticamente com base em regras

<Warning>
  Prefira atribuir acessos a grupos em vez de utilizadores individuais para facilitar a gestão.
</Warning>

### Aplicações e Registo de Aplicações

O Entra ID permite registar e integrar aplicações para autenticação e autorização. Existem dois conceitos relacionados:

1. **Registos de Aplicações**: Representam a definição da aplicação e suas configurações
2. **Aplicações Empresariais**: Representam a instância da aplicação no seu tenant
